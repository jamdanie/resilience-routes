# Contributing

## Before starting

1. Check the GitHub Issues list.
2. Assign the issue to yourself.
3. Confirm the expected file and acceptance criteria.
4. Create a branch from the latest `main`.

## Branch names

Use one of these patterns:

- `feature/short-description`
- `content/short-description`
- `fix/short-description`
- `docs/short-description`

## Pull requests

A pull request should:

- address one focused issue
- explain what changed
- include a screenshot for visual changes
- include sources or notes for real-world scenario claims
- pass `npm run build`
- avoid unrelated formatting changes

## Review checklist

- Does the change support a stated learning objective?
- Is the game still simple enough for a first-time player?
- Are instructions clear?
- Is the content accurate and attributable?
- Are all art and media original or licensed?
- Are there any credentials, personal data, or unsafe HTML patterns?
